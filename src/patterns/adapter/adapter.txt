
Паттерн "Adapter" (Адаптер) - это структурный паттерн проектирования, который позволяет объектам с несовместимыми интерфейсами работать вместе.

Он позволяет обернуть один интерфейс в другой, делая так, чтобы объекты с различными интерфейсами могли взаимодействовать друг с другом.

Введение в паттерн Adapter:
--------------------------

Предположим, у вас есть система электронной коммерции, которая работает с собственным интерфейсом для обработки платежей,
а вы хотите интегрировать стороннюю платежную систему с уже существующим кодом.

Платежная система предоставляет свой собственный интерфейс, который несовместим с вашей системой.

В этом случае паттерн "Адаптер" позволит вам создать адаптер, который адаптирует интерфейс платежной системы к интерфейсу вашей системы.

Основные компоненты паттерна Adapter:
-------------------------------------

Целевой интерфейс:
-----------------------------

Интерфейс, который описывает действия, которые клиент может вызывать.


Адаптируемый класс:
-------------------------------

Существующий класс, у которого есть несовместимый интерфейс.


Адаптер:
------------------

Класс, который реализует целевой интерфейс и оборачивает адаптируемый класс.
Он переводит вызовы методов целевого интерфейса на вызовы методов адаптируемого класса.
